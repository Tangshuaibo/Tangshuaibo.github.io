    使用Vue踩到的坑

    唐帅博 4/26/2020 vue

    # 1. 组件内引用文件夹public内的静态资源（vue2）

    这里根据vue.config.js内选项publicPath的值分类讨论

    # 1.1 publicPath为默认值'/'

    即publicPath为'/'。则vue组件内的路径写法如下：

    <img src="/my-image.png">

    .example {
    background-image: url(/my-image.png);
}

    # 1.2 publicPath被修改，但不使用alias

    例如，publicPath被修改为'/deploymentPath'。则vue组件内的路径写法如下：

    <img :src="`${publicPath}/my-image.png`">

    data () {
  return {
    publicPath: process.env.BASE_URL
  }
}

    <style lang="stylus" scoped>

$publicPath = '/deploymentPath';

.example {
    background-image: url($publicPath + 'my-image.png');
}

<style>

    该方法使用了stylus的语法，需要使用stylus-loader。同理可以使用sass/less等预处理器。

    # 1.3 publicPath被修改，且配置了alias

    例如，vue.config.js进行了如下修改：

    let path = require('path')
...
module.exports = {
    ...
    publicPath: '/deploymentPath',
    ...
    configureWebpack: {
        resolve: {
            alias: {
                '@publicPath': path.resolve(__dirname, './public')
            }
        }
    }
    ...
}

    vue组件内的路径写法如下：

    <img src="@publicPath/my-image.png">

    .example {
    background-image: url('~@publicPath/my-image.png');
}

    1. '@'前缀符号只是用来提示该变量是别名，也可用'$'等符号代替；
    2. VueCLI默认设置了一个指向<projectRoot>/src的别名'@'(仅作用于模版中)。

    1. vue组件内有<img src="~@publicPath/my-image.png">。它的'~@publicPath'中符号'~'可有可无，但是<style>中'background-image: url('~@publicPath/my-image.png');'的'~'符号必须要写；
    2. 写上'~'意味着，要求webpack去解析模块，而不是css-loader（或者可以说，前面加 ~ 表示后面的值为 alias, 然后就会去 webpack alias 配置中找相应的值, 然后拼接成最后的地址）；
    3. 使用这种方式引入public文件夹内的静态资源，会导致打包后的dist文件夹内有两个my-image.png，分别为my-image.png和my-image.[hash].png。

    # 1.4 最佳使用方式

    先说结论：尽量减少使用public文件夹引入静态资源，而多使用assets文件夹。

    VueCLI文档已经指出 (opens new window)：

    public目录提供的是一个应急手段，当你通过绝对路径引用它时，留意应用将会部署到哪里。如果你的应用没有部署在域名的根部，那么你需要为你的 URL 配置 publicPath前缀：

    • 在public/index.html或其它通过html-webpack-plugin用作模板的HTML文件中，你需要通过<%= BASE_URL %>设置链接前缀：
    <link rel="icon" href="<%= BASE_URL %>favicon.ico">
    • 在模板中，你首先需要向你的组件传入基础 URL：
    data () {
  return {
      publicPath: process.env.BASE_URL
  }
}

    然后：

    <img :src="`${publicPath}my-image.png`">

    何时使用 public 文件夹 (opens new window)：

    • 你需要在构建输出中指定一个文件的名字。
    • 你有上千个图片，需要动态引用它们的路径。
    • 有些库可能和webpack不兼容，这时你除了将其用一个独立的<script>标签引入没有别的选择。

    说白了，建议使用public文件夹存放一些不适合webpack打包的第三方插件/库/图片。

    # 2. vue-router重定向时显示错误信息（vue2）

    例如，从/home跳转到/settings，然后在beforeEnter时，利用next('/login')由/settings重定向到/login，控制台显示错误信息Uncaught (in promise) undefined。 需要修改/home内的跳转代码为：this.$router.push('/settings').catch(() => {})

    # 3. $refs无法通过typescript校验（vue2）

    vue-property-decorator有装饰器@Ref

    # 4. eslint+prettier对ts文件不起作用（不提示/不能格式化）（vue2）

    参考文章：

    • 记录vue-cli3.0+typescript项目：Eslint+prettier一些报错以及解决方法（结尾问题已解决）
    • Configuring ESLint (opens new window)
    • ESLint Plugin TypeScript (opens new window)
    • 关键是在parserOptions内添加parser: '@typescript-eslint/parser'
    • 可能与vscode版本有关，由于prettier配置繁琐，且可能不能自动格式化代码
    • 暂时放弃使用prettier，直接用eslint+standard
