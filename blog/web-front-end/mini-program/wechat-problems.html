    关于微信小程序常见的基础问题

    关于微信小程序常见的基础问题

    唐帅博 3/10/2021 mini-program

    # 0. 微信登录接口

    • wx.login将可直接换取unionID
    • wx.getUserInfo将不再返回未加密信息：https://developers.weixin.qq.com/community/develop/doc/000cacfa20ce88df04cb468bc52801
    • 仅获取用户公开信息，可用getUserProfile代替getUserInfo：https://developers.weixin.qq.com/community/develop/article/doc/00062eab0508684539cba249f59413

    # 1. 微信登陆页

    • wx.checkSession：判断用户登录状态是否有效
    • wx.login：获取用户code
    • wx.request：上传code至开发者服务器，返回开发者自定义的登录状态信息（不应将session_key返回用户端），存入localStorage
    • wx.getUserProfile：获取用户头像昵称等（getUserProfile不一定要在login后执行）
    • 以上过程可自己写在登录页面，也可使用微信小程序官方插件：用户信息功能页 (opens new window)

    # 2. 获取加密的用户信息

    • 2021.03后的流程：

      • wx.getUserInfo：直接返回加密信息；解密密文信息，需要wx.login并上传至开发者服务器

    • 2021.03前的流程：

      • 用户点击<button open-type="getUserProfile">
      • wx.getSetting：判断scope.userInfo真值
      • wx.authorize：弹窗授权scope.userInfo
      • wx.openSetting: 用户第一次拒绝授权后，需要通过打开设置页，引导用户再次授权（需要<button open-type="openSetting">触发）
      • wx.getUserInfo：获得不敏感明文信息，以及敏感密文信息；解密密文信息，需要wx.login并上传至开发者服务器

    # 3. 获取手机号（仅对非个人开发者，且完成了认证的小程序开放）

    • 用户点击<button open-type="getPhoneNumber">
    • wx.checkSession：判断用户登录状态是否有效
    • wx.login：获取用户code
    • wx.request: 上传code至开发者服务器，返回新的登陆状态信息，存入localStorage
    • wx.getPhoneNumber：获取加密的用户手机号信息，上传至开发者服务器解密
    • 若只获取加密的手机号信息，则不需要上传服务器解密

    # 4. 微信支付

    • 微信支付文档较为混乱，v3文档主要有两个地址（只看微信支付官网的文档即可）：

      • 在微信支付官网首页可见的：https://pay.weixin.qq.com/wiki/doc/apiv3/wxpay/pages/index.shtml
      • 由小程序开发文档跳转到的：https://pay.weixin.qq.com/wiki/doc/apiv3/wxpay/pages/transactions.shtml

    • 支付流程文档（小程序文档跳转到的）：https://pay.weixin.qq.com/wiki/doc/apiv3/wxpay/pay/transactions/chapter2_3.shtml

    • 小程序向开发者服务器请求生成平台订单（后端向微信服务器请求时，需要openid/公众号id等，即小程序这边需要用户登录）

    • 开发者服务器向微信服务器请求生成微信订单：https://pay.weixin.qq.com/wiki/doc/apiv3/wxpay/pay/transactions/chapter2_3.shtml

    • 开发者服务器返回必要订单数据给前端

    • 小程序调起支付（两种方式：交给后端直接请求微信url与调用wx.requestPayment）：https://pay.weixin.qq.com/wiki/doc/apiv3/wxpay/pay/transactions/chapter3_12.shtml

    • wx.requestPayment：https://developers.weixin.qq.com/miniprogram/dev/api/open-api/payment/wx.requestPayment.html

    • 一个小程序只能绑定一个商户号（直连商户模式），但申请成为微信支付服务商（服务商模式）后可解除限制

      • https://pay.weixin.qq.com/wiki/doc/apiv3/terms_definition/chapter1_1.shtml
      • https://developers.weixin.qq.com/community/develop/doc/000c2474fd0fb0d372f8f7e315b800

    • 微信支付中 paySign 的值是小程序端自己生成还是后端返回?[https://developers.weixin.qq.com/community/pay/doc/000cc80f23c3f0ec8aeaf53a651400]

    • 微信支付paySign在服务端都生成,还是在小程序端生成好?哪种更安全?[https://developers.weixin.qq.com/community/develop/doc/00008a00e3461862d3dae4ed752000]

    • 微信小程序服务商模式支付巨坑解决！[https://blog.csdn.net/qq_18881987/article/details/94428628]

    # 5. 小程序下载

    • https://blog.csdn.net/qq_43568746/article/details/107656279
    • https://developers.weixin.qq.com/miniprogram/dev/framework/ability/file-system.html
    • 微信为每个小程序提供单独的文件夹，主要在 wxafile 和 wxanewfiles 目录下
    • wx.downloadFile和wx.saveFile下载的文件，默认在 wxafile 文件夹内
    • 而使用 wx.env.USER_DATA_PATH 自定义下载路径后，下载的文件在 wxanewfiles 文件夹内
    • 该功能更倾向于持久缓存，而非文件下载。
