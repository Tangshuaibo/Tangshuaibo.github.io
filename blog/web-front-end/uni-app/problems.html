    使用uni-app踩到的坑

    唐帅博 3/10/2021 uni-app

    # 1. subNVue

    • 在pages.json中只能用于APP
    • 在pages.json中只能用于单独页面，globalStyle中无效

    # 2. titleNView

    • 字体只能放在static文件夹下

    # 3. 组件

    • 推荐使用 uView-ui
    • uni-ui可通过npm安装，在typescript项目中引入，需要详细地址，如：import UniIcons from '@dcloudio/uni-ui/lib/uni-icons/uni-icons.vue'
    • $ref：非H5端只能用于获取自定义组件，不能用于获取内置组件实例（如：view、text）
    • Style绑定：非H5端不支持 Vue官方文档中 Class 与 Style 绑定 中的 classObject 和 styleObject 语法
    • uni.createSelectorQuery().in(this).select('active')：在小程序只能取得当前组件内有active类的元素，无法取得子组件内有active类的元素

    # 4. eslint

    • 默认uniapp-ts模板没有eslint校验，直接运行vue-cli的命令以添加：npx vue add eslint
    • 或者可用vue-cli新建vue-ts项目，然后对照其package.json修改uniapp-ts的package.json，并将.eslintrc.js与.editorconfig文件复制过来
    • 若eslint无法检查对*.ts文件，则需要在.eslintrc.js文件的parseOptions添加parser: '@typescript-eslint/parser'

    # 5. 使用uView

    • npm安装uView，但其不支持typescript
      • 参考文章：https://www.typescriptlang.org/docs/handbook/modules.html#ambient-modules
      • 在sfc.d.ts添加如下代码：
        declare module 'uview-ui' {
  import uView from 'uView-ui'
  export default uView
}
        或者其简写：
        declare module 'uVuew-ui'

    # 6. 微信开发者工具预览

    • 当前微信基础库版本过低：
      • 详情 -> 本地设置 -> 调试基础库
    • Cannot read property 'forceUpdate' of undefined：
      • 参考文章：https://ask.dcloud.net.cn/question/101770
      • 即需要在项目manifest.json中添加微信小程序AppID
    • TypeError: Cannot read property 'mark' of undefined：
      • 参考文章：https://blog.csdn.net/weixin_48302334/article/details/110951039
      • 详情 -> 本地设置 -> 增强编译
    • ...toJSON...循环引用：
      • typtescipt编译后，在小程序内预览出错
      • 暂未找到解决方法，只能暂时先使用javascript

    # 7. 页面跳转无反应

    • uni.navigateTo的参数url，可以使用绝对路径（如/pages/index/index）和相对路径（如../home/home）
    • 使用路径pages/index/index无效

    # 8. vue过渡动画在小程序不生效

    • 参考文章：https://uniapp.dcloud.io/api/ui/animation
    • 支持小程序方式的动画，不支持vue的过渡与动画
    1. 为template中引入的自定义组件添加border，小程序中无效
    • border添加到ShadowDom标签上，就是不显示，原因不明，可以外包一层view
    • 另外1rpx可能无法显示，改为2rpx

    # 10. v-show无效

    • https://ask.dcloud.net.cn/question/69259
    • https://ask.dcloud.net.cn/question/75176
    • https://www.cnblogs.com/gxtfighting/p/12419895.html
    • 以上是两种v-show不起作用的情况
    • 可以使用v-if代替，或直接使用class/style控制显隐

    # 11. :style="stlyeObject"无效

    • 非H5端不支持classObject和styleObject语法，但支持数组语法
    • 可以使用:style="[stlyeObject]"

    # 12. input输入框被顶起，修改与键盘距离

    • 配置cursor-spacing、adjust-position
    • https://blog.csdn.net/weixin_34247299/article/details/88705972

    # 13. hideLoading与showLoading不配对

    • showToast等方法，可能提前导致showLoading被hide

    # 14. setStorage会自动调用showLoading

    # 15. 上传小程序时提示：以下文件没有被打包上传

    • https://ask.dcloud.net.cn/question/83159
    • https://developers.weixin.qq.com/community/develop/doc/00068efd3bc1b0365178374da51c00
    • static 文件夹下的内容不会被编译，直接被拷贝，然后可能被微信开发者工具上传时过滤掉。
