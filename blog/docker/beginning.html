关于Windows系统下Docker的安装

唐帅博 2020-04-30 beginning

比较讽刺的是，2020年7月官方宣布docker-toolbox已经弃用了……溜了溜了，我们还是建个linux服务器/虚拟机，再装个docker吧。

反正打死不用hyper-v，因为看到C盘空间小了就抓耳挠腮、寝食难安，而且虚拟机的读写太伤害作为C盘的那块固态硬盘了吧……

# 1. 安装Docker Desktop

很简单，直接去docker官网下载安装包，一直点击“下一步”就能安装成功了。不过目前Docker Desktop有一些缺点：

  1. 只支持Windows 10 Pro，不支持Windows7/8以及10 Home；
  2. 使用时会启动Hyper-V，导致VMware/VirtualBox不能正常使用；
  3. 只能安装在本地磁盘C，而且软件体积非常大，强迫症难以接受。

# 2. 安装Docker Toolbox

我们通过在docker官网的不起眼的小角落里，不停地点击跳转链接，最终找到Docker Toolbox的项目地址和下载地址。

Docker Toolbox默认使用VirtualBox，所以不影响VirtualBox其它虚拟机的使用；它不自动安装启用Hyper-V，软件体积很小，还能自定义安装位置，所以本地C盘还能多剩一点空间。安装教程可以参考这里：win7或win10环境DockerToolbox安装教程。

参考上面这篇文章安装Docker Toolbox时，我们可能还需要注意以下几点：

  1. 最好在安装Docker Toolbox之前就装好Git和VirtualBox客户端，因为可以有更多可以对它们的进行自定义配置的安装选项；
  2. 由于Kitematic客户端在没有科学手段的情况下几乎无法连接到docker官网，同时因为以后更多地使用命令行而非图形客户端控制docker，所以可以选择不安装它；
  3. 点击Finish按钮，就完成安装docker-toolbox了，但是不要直接就去运行Docker Quickstart Terminal。

然后我们进行一些自定义工作：

  1. 找到或添加Window系统下的环境变量MACHINE_STORAGE_PATH，它的值是我们想要安装的docker的linux虚拟机的路径（例如：D:\.docker）;
  2. 把安装目录里的boot2dcoker.iso镜像文件复制到MACHINE_STORAGE_PATH对应路径下的目录\machine\cache\里（例如：D:\.docker\cache\），若文件夹不存在则手动创建；
  3. 其实还可以修改或添加DOCKER_CERT_PATH/DOCKER_HOST/DOCKER_MACHINE_NAME/NO_PROXY等环境变量，不过现在并不推荐去做，因为在创建默认虚拟机时，docker会自动修改它们。

这时我们就能点击图标以运行Docker Quickstart Terminal了，接着docker会自动在MACHINE_STORAGE_PATH指定的路径下安装docker运行所需要的linux虚拟机。

然后我们找到docker的linux虚拟机对应的config.js（路径如：D:\.docker\machines\default\config.json），修改它的RegistryMirror选项为docker国内镜像源，比如https://dockerhub.azk8s.cn（不然docker下载镜像时会连接国外服务器，速度非常慢）。接着命令行运行docker-machine stop default，最后重新双击图标Docker Quickstart Terminal来重启docker的默认虚拟机使配置生效。现在，我们终于能愉快地使用docker了。

附上docker基础教程：Docker 教程 | 菜鸟教程。

  1. 在config.js中修改ResigtryMirror后，直接运行docker-machine restart default是无效的，我们仍需运行Docker Quickstart Terminal以重新配置docker虚拟机内的变量。
  2. 目前大部分docker镜像源已失效，目前我们还可以使用阿里云/腾讯云的镜像加速。以阿里云为例：参考镜像加速器以修改默认镜像，还可以参考访问凭证以修改自定义镜像上传的仓库。
  3. 若执行docker-machine ssh default时返回错误，可能是由于windows系统安装了可选功能OpenSSH客户端；如果不想卸载系统的OpenSSH客户端，那只好每次使用Docker Quickstart Terminal来访问docker虚拟机了。

# 3. 使用Windows10的WSL2

我们可以参考Install Docker Desktop on Windows Home与Docker Desktop WSL 2 backend以配合WSL2使用Docker Desktop。

但是WSL2需要Windows10 2004以上版本支持，且依然使用的是Hyper-V，现阶段仍然会遇到与直接安装Docker Desktop相同的问题，目前尚不推荐。

# 4. 在Linux虚拟机内安装Docker

建议使用virtualbox安装archlinux，因为其软件源内有社区维护的docker本身（非docker镜像源）。相比其他linux发行版，其不用翻墙即可高速下载安装docker。
