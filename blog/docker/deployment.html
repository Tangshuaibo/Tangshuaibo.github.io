    部署Docker容器踩到的坑

    唐帅博 9/2/2020 deployment

    以下内容以在云服务器部署golang服务器为例

    # 1. docker软件本身安装慢

    直接根据docker官网提供的linux端安装教程内的软件（非镜像源）地址，下载缓慢且易断；切换至国内docker软件源下载，发现也不稳定；最终决定使用snap提供的docker包☺️。

    # 2. 运行golang服务器，找不到某些文件

    原因是linux环境与windows环境下文件的换行符不同，从window下复制到linux中的文件，可能无法正常读取。使用dos2unix工具，或使用vim的:set ff=unix命令，以转换相关文件格式。

    # 3. 打包golang服务器镜像，docker镜像的选择

    使用alpine作为基础镜像，那么最好在golang:alpine镜像环境下，将golang源码编译为可执行文件。

    # 4. 等待基础服务完全启动后，再启动golang服务器

    使用wait-for-it等脚本工具

    参考教程：

    • Control startup and shutdown order in Compose (opens new window)
    • vishnubob/wait-for-it (opens new window)

    # 5. 使用let's encrypt开启HTTPS，缺失证书/缺失域名

    • missing certificates：使用let's encrypt启用HTTPS，必须打开80端口，才可获取免费证书。
    • missing server name：使用子域名解析，必须在云服务器的域名解析中添加相关记录。

    # 6. 指定ip的用户无法访问MySQL容器

    • username@container_name：由于命令mysql -u username@container_name -p只是读取字符串，无法做到局域网dns解析。
    • username@specified_ip：
      • 由于docker服务重启后，各容器分配的ip地址可能变动，特别是golang应用容器的ip会变动；
      • 如果不想固定容器ip，那就老实使用通配符（username@"%"）；
      • 在不允许MySQL容器远程连接，或只允许特定用户登录host的情况下，username@"%"已经足够安全了。
