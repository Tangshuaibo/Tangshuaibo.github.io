使用VuePress踩到的坑

唐帅博 2020-04-24 vuepress

# 1. VuePress的全局安装与局部安装

如果一起使用vuepress与vuepress-theme-reco，那么它们的安装位置必须相同。否则，vuepress可能会找不到vuepress-theme-reco中使用的插件，进而报错。

# 2. VuePress的路径要求

vuepress的安装路径不能有特殊符号。例如：路径内有符号&，那么使用npm安装vuepress时会提示部分模块无法安装；路径内有符号{，那么vuepress使用webpack打包时会无法复制静态文件至目标路径，但是没有错误信息。

# 3. Markdown文件内的路径

# 相对路径与绝对路径

我们在这里以引入图片文件至markdown文件做说明。

要在当前markdown文件内显示图片，图片路径可以是相对于目录public/的静态资源路径，如/img.png，也可以是相对于当前目录的路径，如./img.png。值得注意的是，打包后有markdown文件得到的html文件内的图片路径是相对于网站根目录/的绝对路径。

如果我们如果修改了config.js内的配置项base为/blog/，同时还想在markdown文件内使用图片的绝对路径，那么需要把/img.png修改为/blog/img.png。当然，我们也可以用<img :src="$withBase('/img.png')" alt="img.png">来代替![img.png](/blog/img.png)。

如果我们只想在markdown文件内使用图片的相对路径，那么就与是否修改了config.js内的base配置项无关。为了保持工程目录的干净，我们最好在markdown文件所在的文件夹内建立同级的img文件夹，用以提供可用相对路径找到的资源文件。例如，在内容有![img.png](./img/test.png)的markdown文件的所在目录下，建立img/文件夹，用以存放test.png。

另外需要注意的是，路径字符串内不能含有中文，如：各级文件夹、图片的名称等，否则vuepress会运行出错。

# 配置Webpack使用别名

如果要在markdown文件中引用.vuepress/public中的图片img.png，除了可以写为<img :src="$withBase('/img.png')" alt="img.png">，还可以写成![img.png](~@public/img.png)。

只需要按照如下方式修改.vuepress/config.js：

最好不要使用配置别名的方式引入.vuepress/public中的资源，因为打包后public中的静态资源会被简单的复制，同时又被webpack复制并处理了一次，于是打包的最终文件中会有冗余的静态资源。建议新建文件夹.vuepress/assets用于存放静态资源，并对该路径配置别名@assets。

let path = require('path')
module.exports = {
    ...
    configureWebpack: {
        resolve: {
            alias: {
                '@public': path.resolve(__dirname, './public')
            }
        }
    }
    ...
}
  1. 上述代码中的变量__dirname表示的是配置文件config.js所在目录.vuepress在系统中的绝对路径；
  2. 不同于VueCLI默认设置了一个指向<projectRoot>/src的别名@，VuePress并没有默认设置指向.vuepress的别名。

# 4. Stylus文件内的路径

博客部署路径base为默认值/时，样式文件内的图片路径可以使用/或./，如下例：

.sidebar {
    background-image: url(./sidebar_background.png);
}

当base被修改后，如果我们想在样式文件内使用相对于base（如/blog/）的相对路径，那么必须配合使用webpack的配置项alias。

例如，开发环境下的图片文件放在文件夹.vuepress/assets内，那么按照如下方式修改.vuepress/config.js：

let path = require('path')
module.exports = {
    ...
    configureWebpack: {
        resolve: {
            alias: {
                '@assets': path.resolve(__dirname, './assets')
            }
        }
    }
    ...
}

接着在.vuepress/styles/index.styl文件内添加如下代码：

.sidebar {
    background-image: url('~@assets/sidebar_background.png');
}

最后webpack就会在打包时将~@assets替换为相应的路径字符串。

一定要用引号包围路径字符串，如：url('~@assets/sidebar_background.png')，否则webpack会找不到模块。
