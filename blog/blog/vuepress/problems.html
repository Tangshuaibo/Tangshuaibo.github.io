            使用VuePress踩到的坑

            唐帅博 2020-04-24 vuepress

            # 1. VuePress的全局安装与局部安装

            如果一起使用vuepress与vuepress-theme-reco，那么它们的安装位置必须相同。否则，vuepress可能会找不到vuepress-theme-reco中使用的插件，进而报错。

            # 2. VuePress的路径要求

            vuepress的安装路径不能有特殊符号。例如：路径内有符号&，那么使用npm安装vuepress时会提示部分模块无法安装；路径内有符号{，那么vuepress使用webpack打包时会无法复制静态文件至目标路径，但是没有错误信息。

            # 3. Markdown文件内的路径

            我在这里以引入图片文件至markdown文件做说明。

            要在当前markdown文件内显示图片，图片路径可以是相对于目录public/的静态资源路径，如/img.png，也可以是相对于当前目录的路径，如./img.png。值得注意的是，打包后有markdown文件得到的html文件内的图片路径是相对于网站根目录base的绝对路径。

            如果我们如果修改了config.js内的配置项base为/blog/，同时还想在markdown文件内使用图片的绝对路径，那么需要把/img.png修改为/blog/img.png。当然，我们也可以用<img :src="$withBase('/img.png')" alt="img.png">来代替![img.png](/blog/img.png)。

            如果我们只想在markdown文件内使用图片的相对路径，那么就与是否修改了config.js内的base配置项无关。为了保持工程目录的干净，我们最好在markdown文件所在的文件夹内建立同级的img文件夹，用以提供可用相对路径找到的资源文件。例如，在内容有![img.png](./img/test.png)的markdown文件的所在目录下，建立img/文件夹，用以存放test.png。

            # 4. 配置Webpack使用别名

            如果要在markdown文件中引用.vuepress/public中的图片img.png，除了可以写为<img :src="$withBase('/img.png')" alt="img.png">，还可以写成![img.png](~@public/img.png)。

            只需要按照如下方式修改.vuepress/config.js：

            let path = require('path')
module.exports = {
    ...
    configureWebpack: {
        resolve: {
            alias: {
                '@public': path.resolve(__dirname, './public')
            }
        }
    }
    ...
}

            # 提示

            上述代码中的变量__dirname表示的是配置文件config.js所在目录.vuepress在系统中的绝对路径。

            # 注意

            修改configureWebpack.resolve.alias后，我们并不能将别名~@public正确应用在.vuepress/styles/index.styl文件中。

            例如在index.styl文件内添加如下代码：

            .sidebar {
    background-image: url(~@public/sidebar_background.png)
}

            接着运行vuepress后，我们会得到错误信息，vuepress并不能找到index.styl使用的图片文件。

            至于如何在index.styl文件中使用正确的路径信息引入图片，我还在寻找答案。
